## The Two Phases

### Phase 1: CATFIGHT (Understanding)
**Goal:** Deeply understand the problem before writing a single line of code.

This is NOT:
- "Search for tutorials and paste them"
- "Find a DOOM clone and copy it"
- "Give the answer to the coding model"

This IS:
- Reasoning models explaining what they find
- Science models pulling deeper data from technical sources
- Image models doing OCR on diagrams, extracting knowledge from figures
- **Working with the user to distill and refine the objective**

The catfight isn't models fighting each other. It's models **collaborating to understand**.

### Phase 2: SUMMIT (Implementation)
**Goal:** Capture the flag with a refined, high-speed vector.

Only after Phase 1 produces:
- Clear acceptance criteria
- Understood constraints
- Refined architecture
- User-approved direction

Then and only then do the coding models engage.

## The DOOM Connection

There is extensive research on "building DOOM as a software learning experience":
- Fabien Sanglard's "Game Engine Black Book: DOOM"
- Bisqwit's "Creating a DOOM-style 3D engine in C"
- Numerous "build your own raycaster" tutorials
- Academic papers on BSP trees, portal rendering, etc.

**The cats don't just fetch these links.**

The cats:
1. **Read** the sources (science models)
2. **Explain** what they mean (reasoning models)
3. **Extract** diagrams and figures (vision models)
4. **Present** synthesized understanding to the user (ATC display)
5. **Refine** the objective through dialogue
6. **Repeat** until the vector is locked

## The ATC Display Role

The ATC isn't just for watching agents work. It's for:
1. **Presenting synthesized research** - figures, diagrams, summaries
2. **Showing decision points** - "We found 3 approaches, here they are"
3. **Enabling user refinement** - "Which direction resonates?"
4. **Tracking understanding progress** - "We've covered X, Y remains unclear"

The user sees what the agents understand. Not just what they're doing.

## What the Cats Must Learn Before Coding

Before a single `<canvas>` tag:
- [ ] How does raycasting actually work? (not just "it casts rays")
- [ ] What is a BSP tree and why did Carmack use it?
- [ ] How does texture mapping work at the math level?
- [ ] What are the performance constraints of browser JS?
- [ ] How did original DOOM handle level geometry?
- [ ] What can SVG render efficiently vs. what kills performance?
- [ ] How do Rat King patterns tile? Do they tile?

Each question answered by reasoning models. Each answer presented to user. Each confirmation before proceeding.
